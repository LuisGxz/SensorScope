# SensorScope — Estado de fases

> Monitoreo IoT industrial: ingesta de telemetría, gráficos de series temporales en vivo y alertas por
> umbrales. "Sala de control". Leer al iniciar sesión de trabajo.
> Vara de calidad: **FinPulse** (NO NEGOCIABLE — PORTFOLIO_PROJECTS.md §2 "Quality gate"), incl. **Capa de Demo Guiada**.
> **Deploy diferido**: se construye completo menos el deploy en la nube (en bloque al final).

## Stack
- **Front:** Angular 20 (standalone + signals) + Tailwind v4 + **ApexCharts** (series temporales en vivo). **Dark-first** "sala de control".
- **Back:** .NET 9 Web API (Clean Architecture) + **SignalR** (datos en vivo) + motor de evaluación de umbrales.
- **DB:** **TimescaleDB** (PostgreSQL + extensión time-series; `readings` como hypertable). Vía Docker.
- **Auth:** **API keys** (ingesta de dispositivos) + **JWT** (operadores). RBAC simple.
- **Deploy:** Docker + Azure/Render — diferido.

## Cómo correr (dev)
1. TimescaleDB: `docker compose up -d timescaledb` (puerto 5433).
2. `cd backend && dotnet run --project src/SensorScope.Api` → API en `http://localhost:5192` (`/health`). Migra + crea hypertable + seedea al arrancar.
3. `cd frontend && npx ng serve` → app en `http://localhost:4200` (login con cuenta demo).
